Type-2 テスト進捗（2025年6月16日）

ステップ1：

CP2102 USB-UART変換器を通してESP32-CAMをPCに接続
→ 失敗

  • 問題：

    • 問題： PCが新しいデバイスとして認識しなかった

  • 考察：

    • CP2102のドライバがPCにインストールされていない

    ステップ2：

    再度CP2102でESP32-CAMをPCに接続
    → 失敗

  • 問題：

    • デバイスは検出されたが、接続できなかった

  • 考察：

    • CP2102がUSB-TTLモードに設定されていなかった

    ステップ3：

    再度CP2102でESP32-CAMをPCに接続
    → 成功

    ステップ4：

    【テスト20250616-1】を書き込み、距離センサーの接続を確認
    → 失敗

  • 考察：

    • ESP32-CAMはGPIO0をGNDに接続した状態でリセットボタンを押すことで書き込みモードに入る必要がある

    ステップ5：

    【テスト20250616-1】を再度書き込み、距離センサーの接続を確認
    → 失敗

  • 考察：

    • 書き込み後はGPIO0の接続を解除し、再起動する必要がある

    ステップ6：

    【テスト20250616-1】を再度書き込み、距離センサーの接続を確認
    → 失敗

  • 問題：

    • RCWL-1655からの戻り値が一回限りで、かつ内容が文字化けしていた

  • 考察：

    • やはりRX_TRIG_SCL_IOピンをESP32-CAMに接続する必要がある

    ステップ7：

    【テスト20250616-2】を書き込み、距離センサーの接続を確認
    → 成功

    ステップ8：

    【テスト20250616-3】を書き込み、Bluetoothによる接続テストを実施
    → 失敗

    ステップ9：

    【テスト20250616-4】を書き込み、Wi-Fiによる接続テストを実施
    → 一部成功

  • 問題：

    • iPadにてESP32-CAMのデータストリームを受信することに成功。しかし、購入したカメラがESP32-CAMに対応しておらず、プロジェクトは一時中断状態。

    ステップ10：

    【テスト20250616-5】を書き込み、動作パラメータの調整でカメラ互換性を試みる
    → 失敗
